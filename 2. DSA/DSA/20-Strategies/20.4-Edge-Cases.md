# 20.4 Edge Cases

Edge cases are the silent killers of otherwise-correct solutions. A solution that works on the example input but fails on null, empty, overflow, or disconnected inputs will cost you the problem. This file catalogs every recurring edge-case category with concrete checks and code patterns.

## Why Edge Cases Fail Candidates

Edge cases are not "tricks" — they reveal whether you truly understand the problem constraints. Interviewers often deliberately choose test cases that probe boundaries. Missing them signals that your solution is not production-ready.

**The three-step habit:**
1. After coding, mentally run your solution on 3–5 boundary inputs before claiming you're done.
2. Check the constraint table in the problem — minimum values (`0`, `1`, `-10⁹`) are always test cases.
3. Ask yourself: "What happens if the answer is the first element? The last? Every element?"

## Null and Empty Inputs

### Linked List

```java
// Always check head == null before any .next access
ListNode reverseList(ListNode head) {
    if (head == null || head.next == null) return head;  // ← guards
    // ...
}
```

### Tree

```java
// Standard recursive guard
int height(TreeNode root) {
    if (root == null) return 0;                    // ← base case doubles as null guard
    return 1 + Math.max(height(root.left), height(root.right));
}
```

### Array / String

```java
// Check length before indexing
int maxProduct(int[] nums) {
    if (nums == null || nums.length == 0) return 0;  // ← explicit null + empty
    // ...
}

// String
boolean isPalindrome(String s) {
    if (s == null || s.isEmpty()) return true;  // empty string is trivially a palindrome
    // ...
}
```

### Collections

```java
List<Integer> topK(int[] nums, int k) {
    if (nums == null || nums.length == 0 || k <= 0) return Collections.emptyList();
    // ...
}
```

## Single-Element Inputs

Many algorithms break down when n = 1 because the "left neighbor" or "right neighbor" does not exist.

```java
// Sliding window — k > n is a common missed case
double findMaxAverage(int[] nums, int k) {
    if (k > nums.length) return 0;  // or throw, per problem spec
    // ...
}

// Two-pointer pair-sum — single element means no pairs
int[] twoSum(int[] nums, int target) {
    if (nums.length < 2) return new int[]{};
    // ...
}

// Tree: leaf nodes (left == null && right == null)
boolean hasPathSum(TreeNode root, int target) {
    if (root == null) return false;
    if (root.left == null && root.right == null) return root.val == target;  // leaf check
    // ...
}
```

## Integer Overflow

Java's `int` holds up to ~2.1 × 10⁹. Multiplications and sums that cross this boundary silently wrap.

### Multiplication Overflow

```java
// Wrong: int * int can overflow
long product = (long) a * b;        // cast before multiply

// Reverse integer (LC 7) — check before building result
int reverse(int x) {
    long rev = 0;
    while (x != 0) {
        rev = rev * 10 + x % 10;
        x /= 10;
    }
    if (rev > Integer.MAX_VALUE || rev < Integer.MIN_VALUE) return 0;
    return (int) rev;
}
```

### Sum Overflow

```java
// Binary search midpoint — classic overflow bug
int mid = lo + (hi - lo) / 2;      // correct
// int mid = (lo + hi) / 2;        // WRONG if lo + hi > Integer.MAX_VALUE

// Fibonacci / combinatorics — use long or BigInteger
long fib(int n) { /* use long dp[] */ }
```

### Modular Arithmetic

```java
// When the answer must be returned modulo 10^9 + 7
final int MOD = 1_000_000_007;
long result = (a % MOD * (b % MOD)) % MOD;  // multiply first, then mod
```

### Division by Zero / Minimum Integer

```java
// Integer.MIN_VALUE * -1 overflows (still negative)
Math.abs(Integer.MIN_VALUE)  // returns Integer.MIN_VALUE — WRONG!
// Safe check:
if (x == Integer.MIN_VALUE) return 0;  // or handle specially
```

## Off-by-One Errors

### Loop Bounds

```java
// Inclusive vs exclusive — be explicit
for (int i = 0; i < n; i++)       // i goes 0..n-1
for (int i = 0; i <= n; i++)      // i goes 0..n (n+1 iterations)
for (int i = 1; i <= n; i++)      // i goes 1..n
```

### Prefix Sum Indexing

```java
// prefix[i] = sum of first i elements (1-indexed style)
// Sum of nums[l..r] (0-indexed) = prefix[r+1] - prefix[l]
int[] prefix = new int[n + 1];        // size n+1, not n
for (int i = 0; i < n; i++) prefix[i + 1] = prefix[i] + nums[i];
// Query: sum(l, r) = prefix[r + 1] - prefix[l]
```

### Binary Search Boundaries

```java
int lo = 0, hi = nums.length - 1;  // for index search
int lo = 0, hi = nums.length;      // for insertion point (left boundary)

while (lo < hi) {                  // terminates when lo == hi
    int mid = lo + (hi - lo) / 2;
    if (condition(mid)) hi = mid;
    else lo = mid + 1;
}
// lo == hi == answer
```

### Substring / Subarray Length

```java
String sub = s.substring(l, r);          // length = r - l  (r is exclusive)
int[] sub = Arrays.copyOfRange(arr, l, r); // same convention
int windowLength = r - l + 1;            // when r is inclusive (two-pointer style)
```

## Disconnected / Isolated Graph Nodes

### BFS / DFS Must Visit All Components

```java
// Wrong: starts BFS from node 0 only → misses other components
bfs(0, graph);

// Correct: iterate all nodes
for (int i = 0; i < n; i++) {
    if (!visited[i]) bfs(i, graph);
}
```

### Union-Find — Isolated Node Is Its Own Component

```java
// After all union operations, isolated nodes still have parent[i] == i
// Count components:
int components = 0;
for (int i = 0; i < n; i++) if (find(i) == i) components++;
```

### Self-Loop Edges

```java
// Cycle check: a self-loop (u, u) is immediately a cycle
if (u == v) return true;  // or handle before calling union/DFS
```

## Duplicate Values

### Sorted Array With Duplicates — Binary Search

```java
// Finding first / last occurrence requires careful hi handling
int firstOccurrence(int[] nums, int target) {
    int lo = 0, hi = nums.length - 1, result = -1;
    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;
        if (nums[mid] == target) { result = mid; hi = mid - 1; }  // search left
        else if (nums[mid] < target) lo = mid + 1;
        else hi = mid - 1;
    }
    return result;
}
```

### Backtracking — Skip Duplicates at Same Depth

```java
Arrays.sort(nums);
for (int i = start; i < nums.length; i++) {
    if (i > start && nums[i] == nums[i - 1]) continue;  // skip same value at same level
    // ...
}
```

## Negative Numbers and Zero

### Product-Based Problems

```java
// Maximum product subarray (LC 152)
// Negatives flip max↔min — track both
int maxProduct(int[] nums) {
    int max = nums[0], min = nums[0], result = nums[0];
    for (int i = 1; i < nums.length; i++) {
        if (nums[i] < 0) { int tmp = max; max = min; min = tmp; }  // swap on negative
        max = Math.max(nums[i], max * nums[i]);
        min = Math.min(nums[i], min * nums[i]);
        result = Math.max(result, max);
    }
    return result;
}
```

### Zero as Denominator

```java
// Safe division
if (divisor == 0) throw new ArithmeticException("/ by zero");
// Or return a sentinel:
if (b == 0) return Integer.MAX_VALUE;   // "infinite" in some contexts
```

### Zero in Sorted Array (Binary Search)

```java
// The "zero" element can be the answer — initialize result carefully
int lo = 0, hi = n;                // hi = n allows answer = n (all zeros case)
```

### Negative Modulo in Java

```java
// Java's % can return negative for negative operands
int mod = ((x % m) + m) % m;      // always non-negative result
```

## Sorted and Reverse-Sorted Inputs

### Quicksort Degradation

```java
// Naive pivot = first element → O(n²) on sorted input
// Fix: random pivot or median-of-three
int pivotIdx = lo + random.nextInt(hi - lo + 1);
swap(nums, pivotIdx, hi);          // move pivot to end before partitioning
```

### Binary Search — Already Sorted

Always confirm the array is (or can be assumed) sorted before using binary search. If the problem doesn't state sorted, sort first.

### Sliding Window — All Same Values

```java
// All elements equal the target — window may expand to entire array
// Ensure the shrink condition handles equality correctly:
while (windowSum > target) left++;  // shrinks when exceeds, not just equals
```

## Cycle Detection in Graphs

### Directed Graph — Three-Color DFS

```java
// 0 = unvisited, 1 = in-stack (gray), 2 = done (black)
int[] color = new int[n];

boolean hasCycle(int node) {
    color[node] = 1;                           // mark in-stack
    for (int neighbor : graph.get(node)) {
        if (color[neighbor] == 1) return true; // back edge → cycle
        if (color[neighbor] == 0 && hasCycle(neighbor)) return true;
    }
    color[node] = 2;                           // done
    return false;
}
```

### Undirected Graph — Avoid Parent

```java
boolean hasCycleUndirected(int node, int parent, boolean[] visited) {
    visited[node] = true;
    for (int neighbor : graph.get(node)) {
        if (!visited[neighbor]) {
            if (hasCycleUndirected(neighbor, node, visited)) return true;
        } else if (neighbor != parent) {  // ← must exclude parent, not all visited
            return true;
        }
    }
    return false;
}
```

## String Edge Cases

```java
// Empty string
if (s.isEmpty()) return "";

// Single character
if (s.length() == 1) return s;

// All same characters — palindrome, anagram, window problems
// "aaaa" — ensure freq-map logic doesn't double-count

// Case sensitivity — clarify with interviewer
s = s.toLowerCase();    // if case-insensitive required

// Spaces / non-alphanumeric characters
// "A man, a plan, a canal: Panama" — filter before palindrome check
StringBuilder sb = new StringBuilder();
for (char c : s.toCharArray()) {
    if (Character.isLetterOrDigit(c)) sb.append(Character.toLowerCase(c));
}

// Leading/trailing whitespace
s = s.trim();

// String length exceeds int capacity — use long for index
// (practically never in LeetCode, but mention in production contexts)
```

## Floating-Point Precision

```java
// Never compare doubles with ==
double a = 0.1 + 0.2;
// a == 0.3 → FALSE in IEEE 754

// Use epsilon comparison
final double EPS = 1e-9;
if (Math.abs(a - b) < EPS) { /* equal */ }

// For square root problems — binary search on doubles
double lo = 0, hi = x;
for (int iter = 0; iter < 100; iter++) {   // 100 iterations → 2^-100 precision
    double mid = (lo + hi) / 2;
    if (mid * mid <= x) lo = mid;
    else hi = mid;
}
// Or: integer binary search to avoid float entirely
```

## Edge-Case Checklist

Go through this list mentally before declaring a solution complete:

### Input Validity
- [ ] `null` pointer / empty collection?
- [ ] Length / size = 0 or 1?
- [ ] k > n (window larger than array)?
- [ ] Target not present in the input?

### Numeric Bounds
- [ ] Integer overflow on multiplication or addition?
- [ ] `Integer.MIN_VALUE` negation?
- [ ] Division by zero?
- [ ] Negative modulo returning negative?

### Index / Pointer Safety
- [ ] Off-by-one in loop bounds?
- [ ] Binary search mid chosen correctly (`lo + (hi - lo) / 2`)?
- [ ] Prefix/suffix array size is `n+1`, not `n`?
- [ ] Null check before `.next` on linked list?

### Graph / Tree
- [ ] Disconnected components (BFS/DFS starts from all nodes)?
- [ ] Self-loop edges?
- [ ] Directed vs undirected cycle detection?
- [ ] Single-node tree (leaf handling)?

### Duplicates
- [ ] Sorted array with all-same values?
- [ ] Backtracking skips duplicates at the same recursion level?
- [ ] HashMap value equality vs. key collision?

### String
- [ ] Empty string?
- [ ] Single character?
- [ ] All-same characters?
- [ ] Case and whitespace handling?

## Key Takeaways

- Null and empty are the most commonly missed edge cases — add guards at the very top of every function.
- Integer overflow: always cast to `long` before multiplying, and compute binary search midpoints as `lo + (hi - lo) / 2`.
- Off-by-one: decide once whether your bounds are inclusive or exclusive, and be consistent throughout.
- Disconnected graphs: loop over all nodes, not just node 0, when calling BFS/DFS.
- Duplicate values in backtracking: sort first, then `if (i > start && nums[i] == nums[i-1]) continue`.
- Negative numbers: track both max and min in product problems; use `((x % m) + m) % m` for non-negative modulo.
- Run your solution mentally on: empty input, single element, all-same values, reverse-sorted, maximum n before you submit.
