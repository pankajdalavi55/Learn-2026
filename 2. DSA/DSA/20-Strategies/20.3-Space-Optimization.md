# 20.3 Space Optimization

Reducing space complexity — from O(n²) to O(n), or O(n) to O(1) — can be the difference between a solution that passes and one that hits memory limits. This file covers rolling arrays, in-place algorithms, implicit structures, and other space-reduction techniques.

## Why Space Matters

- LeetCode memory limit is typically 256 MB.
- An `int[10⁶][10⁶]` 2D array = 4 TB — impossible.
- Interview rubrics often ask: "Can you do it in O(1) extra space?"
- Even when memory isn't the bottleneck, smaller working sets improve **cache performance** and reduce constant factors in wall-clock time.

## Rolling Arrays in DP

### 1D DP → Two Variables

When `dp[i]` depends only on `dp[i-1]` and `dp[i-2]`, replace the array with two scalars.

```java
// Fibonacci (O(n) space → O(1))
long fib(int n) {
    if (n <= 1) return n;
    long prev2 = 0, prev1 = 1;
    for (int i = 2; i <= n; i++) {
        long cur = prev1 + prev2;
        prev2 = prev1;
        prev1 = cur;
    }
    return prev1;
}
// Time: O(n) | Space: O(1)
```

```java
// House Robber (LC 198) — O(n) space → O(1)
int rob(int[] nums) {
    int prev2 = 0, prev1 = 0;
    for (int x : nums) {
        int cur = Math.max(prev1, prev2 + x);
        prev2 = prev1;
        prev1 = cur;
    }
    return prev1;
}
// Time: O(n) | Space: O(1)
```

### 2D DP → 1D Rolling Array

When `dp[i][j]` depends only on `dp[i-1][...]` (the previous row), keep just two rows.

```java
// 0/1 Knapsack — O(n*W) space → O(W)
int knapsack(int[] weights, int[] values, int W) {
    int[] dp = new int[W + 1];
    for (int i = 0; i < weights.length; i++) {
        // iterate RIGHT-TO-LEFT to avoid using item i twice
        for (int w = W; w >= weights[i]; w--) {
            dp[w] = Math.max(dp[w], dp[w - weights[i]] + values[i]);
        }
    }
    return dp[W];
}
// Time: O(n*W) | Space: O(W)
```

```java
// Longest Common Subsequence (LC 1143) — O(m*n) → O(min(m,n))
int lcs(String s, String t) {
    if (s.length() < t.length()) return lcs(t, s); // ensure s is longer
    int m = s.length(), n = t.length();
    int[] dp = new int[n + 1];
    for (int i = 1; i <= m; i++) {
        int prev = 0;  // dp[i-1][j-1]
        for (int j = 1; j <= n; j++) {
            int temp = dp[j];  // save before overwrite
            if (s.charAt(i - 1) == t.charAt(j - 1)) dp[j] = prev + 1;
            else dp[j] = Math.max(dp[j], dp[j - 1]);
            prev = temp;
        }
    }
    return dp[n];
}
// Time: O(m*n) | Space: O(n)
```

### Diagonal Rolling (Edit Distance)

```java
// Edit Distance (LC 72) — O(m*n) → O(n)
int editDistance(String s, String t) {
    int m = s.length(), n = t.length();
    int[] dp = new int[n + 1];
    for (int j = 0; j <= n; j++) dp[j] = j;   // base case: delete all of t
    for (int i = 1; i <= m; i++) {
        int prev = dp[0];   // dp[i-1][0]
        dp[0] = i;          // base case: delete all of s up to i
        for (int j = 1; j <= n; j++) {
            int temp = dp[j];
            if (s.charAt(i - 1) == t.charAt(j - 1)) dp[j] = prev;
            else dp[j] = 1 + Math.min(prev, Math.min(dp[j], dp[j - 1]));
            prev = temp;
        }
    }
    return dp[n];
}
// Time: O(m*n) | Space: O(n)
```

## In-Place Algorithms

### Array Reversal / Rotation

```java
// Reverse in-place — O(1) space
void reverse(int[] nums, int left, int right) {
    while (left < right) {
        int tmp = nums[left]; nums[left] = nums[right]; nums[right] = tmp;
        left++; right--;
    }
}

// Rotate array right by k — reverse trick
// [1,2,3,4,5], k=2 → [4,5,1,2,3]
void rotate(int[] nums, int k) {
    k %= nums.length;
    reverse(nums, 0, nums.length - 1);
    reverse(nums, 0, k - 1);
    reverse(nums, k, nums.length - 1);
}
// Time: O(n) | Space: O(1)
```

### Partition (Dutch National Flag)

```java
// Sort colors (LC 75) — 0s, 1s, 2s in one pass
void sortColors(int[] nums) {
    int lo = 0, mid = 0, hi = nums.length - 1;
    while (mid <= hi) {
        if (nums[mid] == 0) swap(nums, lo++, mid++);
        else if (nums[mid] == 2) swap(nums, mid, hi--);
        else mid++;
    }
}
// Time: O(n) | Space: O(1)
```

### In-Place Matrix Operations

```java
// Rotate matrix 90° clockwise (LC 48)
// Step 1: transpose  Step 2: reverse each row
void rotate(int[][] matrix) {
    int n = matrix.length;
    // transpose
    for (int i = 0; i < n; i++)
        for (int j = i + 1; j < n; j++) {
            int tmp = matrix[i][j]; matrix[i][j] = matrix[j][i]; matrix[j][i] = tmp;
        }
    // reverse rows
    for (int[] row : matrix) {
        int l = 0, r = n - 1;
        while (l < r) { int tmp = row[l]; row[l++] = row[r]; row[r--] = tmp; }
    }
}
// Time: O(n²) | Space: O(1)
```

## Implicit Structures (Using Input as Storage)

When you need to mark visited cells or track state, use the input array itself (if mutation is allowed) instead of an auxiliary array.

### Mark-Visited with Sign Flip

```java
// Find all numbers disappeared in array (LC 448)
// Mark index nums[i]-1 as negative to indicate i+1 is present
List<Integer> findDisappearedNumbers(int[] nums) {
    for (int x : nums) {
        int idx = Math.abs(x) - 1;
        nums[idx] = -Math.abs(nums[idx]);  // negate to mark
    }
    List<Integer> result = new ArrayList<>();
    for (int i = 0; i < nums.length; i++) {
        if (nums[i] > 0) result.add(i + 1);
    }
    return result;
}
// Time: O(n) | Space: O(1) extra (output list doesn't count)
```

### Cyclic Sort (Find Duplicate / Missing)

```java
// Place each number at index number-1; scan for mismatches
void cyclicSort(int[] nums) {
    int i = 0;
    while (i < nums.length) {
        int correct = nums[i] - 1;
        if (nums[i] != nums[correct]) {
            int tmp = nums[i]; nums[i] = nums[correct]; nums[correct] = tmp;
        } else {
            i++;
        }
    }
}
// Time: O(n) | Space: O(1)
```

### Floyd's Cycle Detection as Implicit Storage

```java
// Find duplicate in array (LC 287) — treat values as pointers
// No extra array, no hash set
int findDuplicate(int[] nums) {
    int slow = nums[0], fast = nums[0];
    do { slow = nums[slow]; fast = nums[nums[fast]]; } while (slow != fast);
    slow = nums[0];
    while (slow != fast) { slow = nums[slow]; fast = nums[fast]; }
    return slow;
}
// Time: O(n) | Space: O(1)
```

## Space-Time Trade-Offs

Sometimes you intentionally **add** space to save time. Know when each direction is appropriate.

| Scenario | Add Space (Cache) | Remove Space (Rolling) |
|----------|------------------|----------------------|
| Repeated subproblem queries | Memoization hash map | — |
| Multiple range queries on static array | Prefix sum array | — |
| DP previous-row only | — | Rolling 1D array |
| DP previous-two values only | — | Two scalar variables |
| Need k-th smallest dynamically | Order-stat tree O(log n) | — |

### Prefix Sum → In-Place Trade

```java
// Grid DP (unique paths) — O(m*n) space → O(n)
int uniquePaths(int m, int n) {
    int[] dp = new int[n];
    Arrays.fill(dp, 1);              // first row: all 1s
    for (int i = 1; i < m; i++)
        for (int j = 1; j < n; j++)
            dp[j] += dp[j - 1];     // dp[j] = from above; dp[j-1] = from left
    return dp[n - 1];
}
// Time: O(m*n) | Space: O(n)
```

## Stack Space (Recursion Depth)

### Why It Matters

Java's default stack depth is ~5 000–10 000 frames. A recursive DFS on a skewed tree of n = 10⁵ nodes causes a `StackOverflowError`.

### Convert Recursion to Iteration

```java
// Iterative inorder traversal (LC 94) — O(h) stack space, not O(n) call stack
List<Integer> inorderTraversal(TreeNode root) {
    List<Integer> result = new ArrayList<>();
    Deque<TreeNode> stack = new ArrayDeque<>();
    TreeNode cur = root;
    while (cur != null || !stack.isEmpty()) {
        while (cur != null) { stack.push(cur); cur = cur.left; }
        cur = stack.pop();
        result.add(cur.val);
        cur = cur.right;
    }
    return result;
}
// Space: O(h) — same order but avoids JVM call-stack limit
```

### Tail-Recursion Elimination

```java
// Tail-recursive factorial (Java does NOT optimize this automatically)
long factorial(int n, long acc) {
    if (n == 0) return acc;
    return factorial(n - 1, n * acc);  // tail call — convert to loop
}

// Iterative equivalent (O(1) stack)
long factorial(int n) {
    long acc = 1;
    while (n > 0) acc *= n--;
    return acc;
}
```

### Morris Traversal (O(1) Space Tree Traversal)

```java
// Inorder traversal without stack or recursion
void morrisInorder(TreeNode root) {
    TreeNode cur = root;
    while (cur != null) {
        if (cur.left == null) {
            visit(cur);
            cur = cur.right;
        } else {
            // find inorder predecessor
            TreeNode pre = cur.left;
            while (pre.right != null && pre.right != cur) pre = pre.right;
            if (pre.right == null) {
                pre.right = cur;    // create thread
                cur = cur.left;
            } else {
                pre.right = null;   // remove thread
                visit(cur);
                cur = cur.right;
            }
        }
    }
}
// Time: O(n) | Space: O(1) — modifies tree temporarily
```

## Bit-Packing

Store multiple boolean or small-integer values inside a single `int` or `long`.

```java
// Sudoku solver: track used digits with bitmask instead of boolean[10]
int[] rowMask = new int[9], colMask = new int[9], boxMask = new int[9];

boolean isUsed(int row, int col, int digit) {
    int bit = 1 << digit;
    int box = (row / 3) * 3 + col / 3;
    return (rowMask[row] & bit) != 0
        || (colMask[col] & bit) != 0
        || (boxMask[box] & bit) != 0;
}

void place(int row, int col, int digit) {
    int bit = 1 << digit;
    int box = (row / 3) * 3 + col / 3;
    rowMask[row] |= bit; colMask[col] |= bit; boxMask[box] |= bit;
}
// 3 int arrays of size 9 replace 3 boolean arrays of size 9*10 = 270 booleans
```

```java
// Subset enumeration with bitmask (2^n subsets of n items)
for (int mask = 0; mask < (1 << n); mask++) {
    // mask uses 1 int instead of n booleans
    for (int i = 0; i < n; i++) {
        if ((mask >> i & 1) == 1) { /* item i is in subset */ }
    }
}
```

## Space Optimization Checklist

1. **DP table**: does `dp[i][j]` depend only on row `i-1`? → collapse to 1D rolling array.
2. **DP row**: does `dp[i]` depend only on `dp[i-1]` and `dp[i-2]`? → two scalar variables.
3. **Visited array**: can you mark visited by negating / XOR-ing the input values?
4. **Auxiliary data structure**: can cyclic sort / Floyd's cycle avoid the HashSet or array?
5. **Recursion depth**: is the tree / graph potentially linear in n? → convert to iterative.
6. **Boolean flags**: can bitmask pack multiple flags into one int?
7. **Coordinate compression**: reduce value range from 10⁹ to n before allocating arrays.

---

## Key Takeaways

- Rolling arrays reduce 2D DP from O(m×n) to O(n) by keeping only the needed row(s).
- Two scalar variables reduce 1D DP to O(1) when only the last two states are needed.
- Knapsack and LCS are the canonical rolling-array patterns to memorize.
- Sign-flip and cyclic sort let you mark state inside the input array with O(1) extra space.
- Floyd's cycle detection can replace a HashSet in duplicate-finding problems.
- Morris traversal gives O(1)-space tree traversal by threading temporary back-pointers.
- Bitmasks pack multiple booleans into a single integer, shrinking both space and constant factors.
- Converting recursion to iteration trades JVM call-stack depth for explicit stack or O(1) space.
