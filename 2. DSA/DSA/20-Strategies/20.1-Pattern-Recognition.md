## Why Patterns Matter

Most interview problems are not novel — they are disguised versions of ~15 core patterns. Once you internalize the trigger signals for each pattern, you spend your mental energy on the interesting part (tweaking the template) rather than deriving an approach from scratch.

**The recognition loop:**
1. Read the problem — identify *what* is being asked (min/max? count? existence? construction?)
2. Look at constraints — array size, value range, sorted vs unsorted
3. Match signals to a pattern
4. Reach for the template, then adapt

## Sliding Window

### When to Use
- Subarray / substring of **fixed or variable length**
- Problem involves a contiguous range of elements
- Brute force is O(n²) but you suspect O(n)

### Trigger Keywords
"longest subarray", "shortest subarray", "subarray with sum/count ≤ k", "minimum window", "at most k distinct"

### Fixed-Window Template

```java
int windowSum = 0;
for (int i = 0; i < k; i++) windowSum += nums[i];     // seed first window
int maxSum = windowSum;
for (int i = k; i < nums.length; i++) {
    windowSum += nums[i] - nums[i - k];                // slide: add right, drop left
    maxSum = Math.max(maxSum, windowSum);
}
// Time: O(n) | Space: O(1)
```

### Variable-Window Template

```java
int left = 0, result = 0;
// state = whatever makes the window valid (sum, freq map, distinct count …)
for (int right = 0; right < n; right++) {
    // expand: include nums[right] in state
    while (/* window invalid */) {
        // shrink: remove nums[left] from state
        left++;
    }
    result = Math.max(result, right - left + 1);
}
// Time: O(n) | Space: O(1) or O(k)
```

## Two Pointers

### When to Use
- **Sorted** array (or can be sorted cheaply)
- Looking for pairs / triplets summing to a target
- Removing duplicates in-place
- Partitioning around a pivot

### Trigger Keywords
"two sum sorted", "remove duplicates", "container with most water", "trapping rain water", "3-sum"

### Opposite-Ends Template

```java
int left = 0, right = nums.length - 1;
while (left < right) {
    int sum = nums[left] + nums[right];
    if (sum == target) { /* found */ }
    else if (sum < target) left++;
    else right--;
}
// Time: O(n) | Space: O(1)
```

### Same-Direction (Fast/Slow Write) Template

```java
int writePtr = 0;
for (int readPtr = 0; readPtr < nums.length; readPtr++) {
    if (/* nums[readPtr] should be kept */) {
        nums[writePtr++] = nums[readPtr];
    }
}
// Result: first writePtr elements are kept
// Time: O(n) | Space: O(1)
```

### Classic Problems
| Problem | LC # | Variant |
|---------|------|---------|
| Two Sum II | 167 | Opposite ends |
| 3Sum | 15 | Fix one, opposite ends |
| Container With Most Water | 11 | Opposite ends |
| Remove Duplicates from Sorted Array | 26 | Same-direction write |
| Trapping Rain Water | 42 | Opposite ends with running max |

## Fast and Slow Pointers

### When to Use
- **Linked list** cycle detection or midpoint finding
- Detect if a sequence is periodic

### Trigger Keywords
"cycle in linked list", "find middle", "happy number"

### Template

```java
ListNode slow = head, fast = head;
while (fast != null && fast.next != null) {
    slow = slow.next;
    fast = fast.next.next;
    if (slow == fast) return true;  // cycle detected
}
return false;
// Time: O(n) | Space: O(1)
```

| Problem | LC # |
|---------|------|

| Linked List Cycle II (start of cycle) | 142 |
| Middle of the Linked List | 876 |
| Happy Number | 202 |

## Prefix Sum

### When to Use
- **Range sum queries** (static array)
- Subarray sum equals / at most / at least k
- Difference arrays for range updates

### Trigger Keywords
"subarray sum", "range sum", "number of subarrays with sum"

### Build & Query Template

```java
int[] prefix = new int[n + 1];
for (int i = 0; i < n; i++) prefix[i + 1] = prefix[i] + nums[i];
// sum of nums[l..r] (inclusive) = prefix[r+1] - prefix[l]
```

### Prefix Sum + HashMap (Count Subarrays)

```java
Map<Integer, Integer> count = new HashMap<>();
count.put(0, 1);
int sum = 0, result = 0;
for (int x : nums) {
    sum += x;
    result += count.getOrDefault(sum - k, 0);
    count.merge(sum, 1, Integer::sum);
}
// Time: O(n) | Space: O(n)
```

|---------|------|
| Range Sum Query (Immutable) | 303 |
| Subarray Sum Equals K | 560 |
| Contiguous Array | 525 |
| Product of Array Except Self | 238 |

## Monotonic Stack / Queue

### When to Use
- **Next greater / smaller element**
- Largest rectangle / maximal area
- Sliding window maximum

### Trigger Keywords
"next greater element", "daily temperatures", "largest rectangle", "sliding window max"

### Monotonic Stack (Next Greater) Template

```java
Deque<Integer> stack = new ArrayDeque<>();  // stores indices
int[] result = new int[n];
Arrays.fill(result, -1);
for (int i = 0; i < n; i++) {
    while (!stack.isEmpty() && nums[stack.peek()] < nums[i]) {
        result[stack.pop()] = nums[i];  // nums[i] is NGE of popped index
    }
    stack.push(i);
}
// Time: O(n) | Space: O(n)
```

### Monotonic Deque (Sliding Window Max) Template

```java
Deque<Integer> deque = new ArrayDeque<>();  // stores indices, front = max
int[] result = new int[n - k + 1];
for (int i = 0; i < n; i++) {
    while (!deque.isEmpty() && deque.peekFirst() < i - k + 1) deque.pollFirst(); // out of window
    while (!deque.isEmpty() && nums[deque.peekLast()] <= nums[i]) deque.pollLast();
    deque.offerLast(i);
    if (i >= k - 1) result[i - k + 1] = nums[deque.peekFirst()];
}
// Time: O(n) | Space: O(k)
```

## Binary Search on Answer

### When to Use
- Answer lives in a **monotonic search space** (bigger k → easier / harder)
- "Minimum of maximum" or "maximum of minimum" phrasing

### Trigger Keywords
"minimum days", "koko eating bananas", "capacity to ship", "split array largest sum"

### Template

```java
int lo = minPossibleAnswer, hi = maxPossibleAnswer;
while (lo < hi) {
    int mid = lo + (hi - lo) / 2;
    if (feasible(mid)) hi = mid;   // mid works → try smaller
    else lo = mid + 1;             // mid too small
}
return lo;  // lo == hi == answer
// Time: O(log(range) * cost_of_feasible)
```

| Capacity To Ship Packages | 1011 |
| Split Array Largest Sum | 410 |
| Minimum Number of Days to Make m Bouquets | 1482 |
| Find the Smallest Divisor | 1283 |

## BFS / Level-Order

### When to Use
- **Shortest path** in unweighted graph / grid
- Level-by-level tree processing
- Multi-source spreading

### Trigger Keywords
"minimum steps", "shortest path", "level order", "01 matrix", "rotten oranges"

### BFS Template

```java
Queue<int[]> queue = new LinkedList<>();
boolean[][] visited = new boolean[rows][cols];
queue.offer(new int[]{startR, startC});
visited[startR][startC] = true;
int steps = 0;
int[][] dirs = {{0,1},{0,-1},{1,0},{-1,0}};

while (!queue.isEmpty()) {
    int size = queue.size();
    while (size-- > 0) {
        int[] cur = queue.poll();
        if (isGoal(cur)) return steps;
        for (int[] d : dirs) {
            int nr = cur[0] + d[0], nc = cur[1] + d[1];
            if (inBounds(nr, nc) && !visited[nr][nc]) {
                visited[nr][nc] = true;
                queue.offer(new int[]{nr, nc});
            }
        }
    }
    steps++;
}
// Time: O(V + E) | Space: O(V)
```

## DFS / Backtracking

### When to Use
- **Enumerate all** solutions (permutations, subsets, paths)
- Explore a graph / grid exhaustively
- Constraint satisfaction (N-Queens, Sudoku)

### Trigger Keywords
"all combinations", "all permutations", "generate all", "word search", "path exists"

### Backtracking Template

```java
void backtrack(List<List<Integer>> result, List<Integer> current, int start, int[] nums) {
    result.add(new ArrayList<>(current));   // record (for subsets: always; for permutations: at base case)
    for (int i = start; i < nums.length; i++) {
        if (/* should skip */) continue;
        current.add(nums[i]);               // choose
        backtrack(result, current, i + 1, nums);  // explore
        current.remove(current.size() - 1);  // un-choose
    }
}
```

| Word Search | 79 |

## Dynamic Programming Signals

### When to Use
- **Overlapping subproblems** + **optimal substructure**
- Count/min/max over decisions, not enumeration
- Problem says "number of ways", "minimum cost", "can you reach"

### Trigger Keywords
"minimum cost", "number of ways", "longest subsequence", "can partition", "decode ways"

### State-Definition Checklist
1. What does `dp[i]` (or `dp[i][j]`) represent?
2. What is the base case?
3. What is the recurrence?
4. What is the answer? (`dp[n]`? `max(dp)`?)

```java
// 1D DP scaffold
int[] dp = new int[n + 1];
dp[0] = BASE_CASE;
for (int i = 1; i <= n; i++) {
    dp[i] = /* recurrence using dp[i-1], dp[i-2], … */;
}
return dp[n];
```

## Greedy Signals

### When to Use
- Local optimal choice leads to global optimum (provable by exchange argument)
- Sorting + one pass yields the answer
- Interval scheduling / activity selection

### Trigger Keywords
"minimum number of", "maximum profit", "earliest finish time", "can complete"

### Exchange Argument Check
Ask: "If I swap a greedy choice with any other choice, does the result get worse or stay the same?" If yes → greedy is correct.

```java
// Interval scheduling (maximum non-overlapping intervals)
Arrays.sort(intervals, (a, b) -> a[1] - b[1]);  // sort by end time
int count = 1, end = intervals[0][1];
for (int i = 1; i < intervals.length; i++) {
    if (intervals[i][0] >= end) {
        count++;
        end = intervals[i][1];
    }
}
// Time: O(n log n) | Space: O(1)
```

## Union-Find

### When to Use
- **Connectivity** in undirected graph
- Dynamic graph where edges are added online
- Minimum spanning tree (Kruskal)

### Trigger Keywords
"connected components", "redundant connection", "accounts merge", "number of provinces"

### Template

```java
int[] parent, rank;

void init(int n) {
    parent = new int[n];
    rank = new int[n];
    for (int i = 0; i < n; i++) parent[i] = i;
}

int find(int x) {
    if (parent[x] != x) parent[x] = find(parent[x]);  // path compression
    return parent[x];
}

boolean union(int x, int y) {
    int px = find(x), py = find(y);
    if (px == py) return false;  // already connected
    if (rank[px] < rank[py]) { int tmp = px; px = py; py = tmp; }
    parent[py] = px;
    if (rank[px] == rank[py]) rank[px]++;
    return true;
}
// Time: O(α(n)) ≈ O(1) per operation
```

## Pattern Decision Tree

```
Is the input a LINKED LIST?
  → Cycle? → Fast/Slow
  → Midpoint/Merge? → Fast/Slow / Two Pointers

Is the input an ARRAY/STRING?
  → Contiguous subarray? → Sliding Window
  → Sorted / pair sum? → Two Pointers
  → Range sum? → Prefix Sum
  → Next greater/smaller? → Monotonic Stack
  → Minimum of max or max of min? → Binary Search on Answer

Is the input a GRAPH/GRID?
  → Shortest path (unweighted)? → BFS
  → Connectivity? → Union-Find / DFS
  → Topo order? → Topological Sort (Kahn's)
  → All paths? → DFS/Backtracking

Is the input a TREE?
  → Level-by-level? → BFS
  → Path sum / LCA / diameter? → DFS post-order
  → BST range? → Inorder

OPTIMIZATION (min/max/count)?
  → Overlapping subproblems? → DP
  → Greedy exchange argument holds? → Greedy
  → Must enumerate all? → Backtracking
```

## Key Takeaways

- Sliding Window → contiguous subarray/substring with O(n) over brute-force O(n²).
- Two Pointers → sorted arrays, pair sums, in-place filtering.
- Fast/Slow → linked list cycles and midpoints.
- Prefix Sum → range queries and subarray sum problems.
- Monotonic Stack → next greater/smaller element, O(n) amortized.
- Binary Search on Answer → "minimum of maximum" over a monotone feasibility function.
- BFS → shortest path (unweighted), level-order, multi-source spreading.
- DP → overlapping subproblems + optimal substructure; define state, base, recurrence.
- Greedy → when local optimum = global optimum; prove with exchange argument.
- Union-Find → dynamic connectivity with near-O(1) amortized operations.
