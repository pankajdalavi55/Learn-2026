# 10.3 Collision Handling

When two keys hash to the same bucket, a collision occurs. How collisions are resolved determines worst-case performance and memory usage — knowing the trade-offs is key to designing robust hash-based systems.

## What is a Collision?

A **collision** occurs when two distinct keys produce the same bucket index.

```
hash("apple")  % 8 = 3
hash("mango")  % 8 = 3  ← collision!

Both keys want bucket[3]
```

### Why Collisions Happen

- Hash function outputs finite range (bucket count), but key space is infinite.
- Birthday paradox: with n=16 buckets, collision probability exceeds 50% after only ~5 insertions.
- Clustering makes it worse with poor hash functions.

## Separate Chaining

### Concept

Each bucket holds a **linked list** (or tree) of all entries that hash to that bucket.

```
Bucket Array (capacity = 7):

bucket[0] → null
bucket[1] → ["cat",3] → ["rat",7] → null
bucket[2] → ["dog",5] → null
bucket[3] → null
bucket[4] → ["bat",2] → null
bucket[5] → ["hat",9] → ["mat",1] → null
bucket[6] → null
```

### Implementation

```java
public class SeparateChainingHashMap<K, V> {
    private static final int DEFAULT_CAPACITY = 16;

    private static class Node<K, V> {
        K key;
        V value;
        Node<K, V> next;

        Node(K key, V value) {
            this.key = key;
            this.value = value;
        }
    }

    @SuppressWarnings("unchecked")
    private Node<K, V>[] buckets = new Node[DEFAULT_CAPACITY];
    private int size = 0;

    private int bucketIndex(K key) {
        return (key.hashCode() & 0x7fffffff) % buckets.length;
    }

    public void put(K key, V value) {
        int idx = bucketIndex(key);
        Node<K, V> head = buckets[idx];

        // Update if key exists
        for (Node<K, V> n = head; n != null; n = n.next) {
            if (n.key.equals(key)) {
                n.value = value;
                return;
            }
        }

        // Prepend new node
        Node<K, V> newNode = new Node<>(key, value);
        newNode.next = head;
        buckets[idx] = newNode;
        size++;
    }

    public V get(K key) {
        int idx = bucketIndex(key);
        for (Node<K, V> n = buckets[idx]; n != null; n = n.next) {
            if (n.key.equals(key)) return n.value;
        }
        return null;
    }

    public void remove(K key) {
        int idx = bucketIndex(key);
        Node<K, V> prev = null, curr = buckets[idx];
        while (curr != null) {
            if (curr.key.equals(key)) {
                if (prev == null) buckets[idx] = curr.next;
                else prev.next = curr.next;
                size--;
                return;
            }
            prev = curr;
            curr = curr.next;
        }
    }

    public int size() { return size; }
}
```

### Complexity: Separate Chaining

| Operation | Average | Worst Case |
|-----------|---------|------------|
| put | O(1) | O(n) — all keys in one bucket |
| get | O(1) | O(n) |
| remove | O(1) | O(n) |
| Space | O(n) | O(n) |

## Open Addressing

### Concept

All entries stored **inside** the bucket array. On collision, probe for the next empty slot.

```
Insert "apple" (hash→3): bucket[3] empty → place here
Insert "mango" (hash→3): bucket[3] full → probe bucket[4]
Insert "grape" (hash→4): bucket[4] full → probe bucket[5]
```

### Linear Probing

```java
public class LinearProbingHashMap<K, V> {
    private Object[] keys;
    private Object[] values;
    private boolean[] deleted;
    private int capacity;
    private int size;

    @SuppressWarnings("unchecked")
    public LinearProbingHashMap(int capacity) {
        this.capacity = capacity;
        keys = new Object[capacity];
        values = new Object[capacity];
        deleted = new boolean[capacity];
    }

    private int hash(K key) {
        return (key.hashCode() & 0x7fffffff) % capacity;
    }

    public void put(K key, V value) {
        int idx = hash(key);
        while (keys[idx] != null && !deleted[idx]) {
            if (keys[idx].equals(key)) {
                values[idx] = value;
                return;
            }
            idx = (idx + 1) % capacity;  // linear probe
        }
        keys[idx] = key;
        values[idx] = value;
        deleted[idx] = false;
        size++;
    }

    @SuppressWarnings("unchecked")
    public V get(K key) {
        int idx = hash(key);
        while (keys[idx] != null) {
            if (!deleted[idx] && keys[idx].equals(key)) {
                return (V) values[idx];
            }
            idx = (idx + 1) % capacity;
        }
        return null;
    }

    public void remove(K key) {
        int idx = hash(key);
        while (keys[idx] != null) {
            if (!deleted[idx] && keys[idx].equals(key)) {
                deleted[idx] = true;  // lazy deletion
                size--;
                return;
            }
            idx = (idx + 1) % capacity;
        }
    }
}
```

### Quadratic Probing

```
Probe sequence: h(k), h(k)+1², h(k)+2², h(k)+3², ...

Step 1: idx = hash(key)
Step 2: if occupied → idx = (hash + i²) % capacity  for i = 1, 2, 3...

Advantage: Reduces primary clustering (long runs of occupied slots)
Disadvantage: Secondary clustering (same hash → same probe sequence)
```

```java
private int probe(int hash, int i) {
    return (hash + i * i) % capacity;
}
```

### Double Hashing

```
Two independent hash functions h1 and h2.
Probe sequence: h1(k), h1(k)+h2(k), h1(k)+2*h2(k), ...

h1(k) = k % capacity
h2(k) = 1 + (k % (capacity - 1))   // must be non-zero and coprime to capacity

Advantage: Eliminates clustering entirely
Best distribution among open addressing methods
```

```java
private int hash1(K key) {
    return (key.hashCode() & 0x7fffffff) % capacity;
}

private int hash2(K key) {
    return 1 + ((key.hashCode() & 0x7fffffff) % (capacity - 1));
}

private int probe(K key, int i) {
    return (hash1(key) + i * hash2(key)) % capacity;
}
```

### Probing Strategy Comparison

```
Primary Clustering (Linear Probing):
[A][B][C][D][_][_]  — occupied runs grow, slowing future ops

Secondary Clustering (Quadratic Probing):
Spread out more, but same h1 → same sequence

Double Hashing:
h2 randomizes step size, no clustering
```

## Load Factor & Rehashing

### Load Factor

```
Load Factor (α) = n / capacity
  n       = number of elements
  capacity = number of buckets

Java HashMap default: capacity=16, loadFactor=0.75
Resize trigger: when n > 16 * 0.75 = 12 elements
```

### Performance vs Load Factor

| Load Factor | Avg Probes (Linear) | Avg Probes (Chaining) |
|-------------|--------------------|-----------------------|
| 0.50 | 1.5 | 1.25 |
| 0.75 | 2.5 | 1.375 |
| 0.90 | 5.5 | 1.45 |
| 1.00 | ∞ | 1.50 |

### Rehashing Process

```java
private void rehash() {
    Node<K, V>[] oldBuckets = buckets;
    int newCapacity = oldBuckets.length * 2;

    @SuppressWarnings("unchecked")
    Node<K, V>[] newBuckets = new Node[newCapacity];
    buckets = newBuckets;
    size = 0;

    for (Node<K, V> head : oldBuckets) {
        for (Node<K, V> n = head; n != null; n = n.next) {
            put(n.key, n.value);  // re-insert into new bucket array
        }
    }
}
```

## Java's HashMap Collision Strategy

```
Java uses Separate Chaining with two tiers:

1. LinkedList tier (≤ 8 entries per bucket):
   - Fast insertion: O(1)
   - Search: O(k) where k = chain length

2. Red-Black Tree tier (> 8 entries per bucket):
   - Treeify threshold = 8
   - Untreeify threshold = 6
   - Search: O(log k) worst case

Capacity always a power of 2 → fast modulo via bit AND:
   index = hash & (capacity - 1)

Spread function to reduce collisions:
   hash = key.hashCode() ^ (key.hashCode() >>> 16)
   (XORs upper 16 bits into lower 16 bits)
```

---

## Comparison of Strategies

| Strategy | Cache Performance | Space | Deletion | Clustering |
|----------|------------------|-------|----------|------------|
| Separate Chaining | Poor (pointer chasing) | Extra nodes | Simple | None |
| Linear Probing | Excellent (sequential) | In-place | Lazy delete | Primary |
| Quadratic Probing | Good | In-place | Lazy delete | Secondary |
| Double Hashing | Moderate | In-place | Lazy delete | None |

**Java HashMap uses Separate Chaining** — simplicity + tree fallback handles worst-case well.

## Custom HashMap Implementation

Full working example combining everything:

```java
public class CustomHashMap<K, V> {
    private static final int INITIAL_CAPACITY = 16;
    private static final float LOAD_FACTOR = 0.75f;
    private static final int TREEIFY_THRESHOLD = 8;

    private Object[] table;
    private int size;
    private int threshold;

    private static class Entry<K, V> {
        final K key;
        V value;
        Entry<K, V> next;
        final int hash;

        Entry(K key, V value, int hash, Entry<K, V> next) {
            this.key = key;
            this.value = value;
            this.hash = hash;
            this.next = next;
        }
    }

    @SuppressWarnings("unchecked")
    public CustomHashMap() {
        table = new Object[INITIAL_CAPACITY];
        threshold = (int)(INITIAL_CAPACITY * LOAD_FACTOR);
    }

    private int hash(K key) {
        int h = key.hashCode();
        return h ^ (h >>> 16);  // spread bits
    }

    private int index(int hash) {
        return hash & (table.length - 1);
    }

    @SuppressWarnings("unchecked")
    public void put(K key, V value) {
        if (size >= threshold) resize();

        int h = hash(key);
        int idx = index(h);
        Entry<K, V> entry = (Entry<K, V>) table[idx];

        for (Entry<K, V> e = entry; e != null; e = e.next) {
            if (e.hash == h && e.key.equals(key)) {
                e.value = value;
                return;
            }
        }

        table[idx] = new Entry<>(key, value, h, entry);
        size++;
    }

    @SuppressWarnings("unchecked")
    public V get(K key) {
        int h = hash(key);
        int idx = index(h);
        for (Entry<K, V> e = (Entry<K, V>) table[idx]; e != null; e = e.next) {
            if (e.hash == h && e.key.equals(key)) return e.value;
        }
        return null;
    }

    @SuppressWarnings("unchecked")
    private void resize() {
        Object[] old = table;
        table = new Object[old.length * 2];
        threshold = (int)(table.length * LOAD_FACTOR);
        size = 0;

        for (Object head : old) {
            for (Entry<K, V> e = (Entry<K, V>) head; e != null; e = e.next) {
                put(e.key, e.value);
            }
        }
    }

    public int size() { return size; }
}
```

## Key Takeaways

- Separate chaining is simple and handles high load factors gracefully; Java uses it with tree fallback.
- Linear probing is cache-friendly but suffers primary clustering; keep α < 0.7.
- Double hashing eliminates clustering but requires a carefully chosen second hash function.
- Java's hash spread (`h ^ h>>>16`) reduces collisions caused by poor `hashCode` implementations.
- Resizing at α = 0.75 is the sweet spot between space waste and collision rate.
- Lazy deletion (mark deleted slots) is required for open addressing; simple chain removal for separate chaining.
