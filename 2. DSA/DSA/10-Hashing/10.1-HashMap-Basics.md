## Introduction

### What is a HashMap?

A **HashMap** is a data structure that maps **keys to values** using a hash function to compute an index into an array of buckets.

```
Key → hash(key) → bucket index → value

"apple" → hashCode() → 3 → bucket[3] → 5
"banana" → hashCode() → 7 → bucket[7] → 2
```

### Why Use HashMap?

| Operation | Array/List | HashMap |
|-----------|-----------|---------|
| Search | O(n) | O(1) avg |
| Insert | O(1) amortized | O(1) avg |
| Delete | O(n) | O(1) avg |
| Lookup by key | O(n) | O(1) avg |

## How HashMap Works Internally

### Structure

```
HashMap<K, V>
     ↓
[ bucket0 ] → Entry(k1,v1) → Entry(k2,v2) → null
[ bucket1 ] → null
[ bucket2 ] → Entry(k3,v3) → null
[ bucket3 ] → Entry(k4,v4) → null
...
[ bucketN ] → null

Default capacity = 16, Load factor = 0.75
```

### Hashing Process

```
Step 1: key.hashCode() → raw hash (32-bit int)
Step 2: spread(hash)   → better distribution (XOR with upper bits)
Step 3: hash % capacity → bucket index

Java 8+:
int hash = (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
int index = hash & (capacity - 1);  // capacity always power of 2
```

### Resizing (Rehashing)

```
When size > capacity * loadFactor (default 0.75):

1. New capacity = 2 × old capacity
2. New array allocated
3. All entries re-hashed into new buckets

Example:
  capacity=16, threshold=12
  After 12 inserts → resize to capacity=32
```

### Java 8 Optimization: TreeMap Buckets

```
Bucket with ≤ 8 entries  → LinkedList (fast insert)
Bucket with > 8 entries  → Red-Black Tree (O(log n) worst case)

Treeify threshold = 8
Untreeify threshold = 6
```

## Core Operations

### put / get / remove

```java
import java.util.*;

public class HashMapBasics {
    public static void main(String[] args) {
        HashMap<String, Integer> map = new HashMap<>();

        // put(key, value) — O(1) avg
        map.put("apple", 3);
        map.put("banana", 5);
        map.put("cherry", 1);

        // get(key) — O(1) avg, returns null if absent
        System.out.println(map.get("apple"));   // 3
        System.out.println(map.get("mango"));   // null

        // getOrDefault — safe alternative to get
        int count = map.getOrDefault("mango", 0);  // 0

        // containsKey / containsValue
        System.out.println(map.containsKey("banana"));    // true
        System.out.println(map.containsValue(99));        // false

        // remove(key) — O(1) avg
        map.remove("cherry");

        // size
        System.out.println(map.size());  // 2

        // putIfAbsent — insert only if key missing
        map.putIfAbsent("apple", 100);   // no change, key exists
        map.putIfAbsent("date", 7);      // inserts

        // replace
        map.replace("apple", 10);

        System.out.println(map);  // {apple=10, banana=5, date=7}
    }
}
```

### merge & compute

```java
HashMap<String, Integer> freq = new HashMap<>();
String[] words = {"a", "b", "a", "c", "b", "a"};

for (String w : words) {
    // merge: if key absent → newValue, else → remapping function
    freq.merge(w, 1, Integer::sum);
}
// {a=3, b=2, c=1}

// compute: always applies function
freq.compute("a", (k, v) -> v == null ? 1 : v + 1);

// computeIfAbsent: only when key is missing
Map<String, List<Integer>> groups = new HashMap<>();
groups.computeIfAbsent("evens", k -> new ArrayList<>()).add(2);
groups.computeIfAbsent("evens", k -> new ArrayList<>()).add(4);
// {evens=[2, 4]}
```

## Iteration Patterns

```java
HashMap<String, Integer> map = new HashMap<>();
map.put("a", 1);
map.put("b", 2);
map.put("c", 3);

// 1. entrySet — most efficient, both key and value
for (Map.Entry<String, Integer> entry : map.entrySet()) {
    System.out.println(entry.getKey() + " → " + entry.getValue());
}

// 2. keySet — when you only need keys
for (String key : map.keySet()) {
    System.out.println(key + " → " + map.get(key));
}

// 3. values — when you only need values
for (int val : map.values()) {
    System.out.println(val);
}

// 4. forEach (Java 8+) — cleanest
map.forEach((k, v) -> System.out.println(k + "=" + v));
```

### Sorted Iteration

```java
// By key (natural order) → use TreeMap
TreeMap<String, Integer> sorted = new TreeMap<>(map);

// By value → sort entrySet
map.entrySet().stream()
   .sorted(Map.Entry.comparingByValue())
   .forEach(e -> System.out.println(e.getKey() + "=" + e.getValue()));

// By value descending
map.entrySet().stream()
   .sorted(Map.Entry.<String, Integer>comparingByValue().reversed())
   .forEach(e -> System.out.println(e.getKey() + "=" + e.getValue()));
```

## Common HashMap Patterns

### Pattern 1: Frequency Count

```java
public Map<Character, Integer> charFrequency(String s) {
    Map<Character, Integer> freq = new HashMap<>();
    for (char c : s.toCharArray()) {
        freq.merge(c, 1, Integer::sum);
    }
    return freq;
}
```

### Pattern 2: Group By

```java
// Group words by their sorted characters (anagram grouping)
public List<List<String>> groupAnagrams(String[] strs) {
    Map<String, List<String>> map = new HashMap<>();
    for (String s : strs) {
        char[] arr = s.toCharArray();
        Arrays.sort(arr);
        String key = new String(arr);
        map.computeIfAbsent(key, k -> new ArrayList<>()).add(s);
    }
    return new ArrayList<>(map.values());
}
```

### Pattern 3: Two-Pass Index Map

```java
// Two Sum (LeetCode 1)
public int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> indexMap = new HashMap<>();
    for (int i = 0; i < nums.length; i++) {
        int complement = target - nums[i];
        if (indexMap.containsKey(complement)) {
            return new int[]{indexMap.get(complement), i};
        }
        indexMap.put(nums[i], i);
    }
    return new int[]{};
}
```

### Pattern 4: Sliding Window with Map

```java
// Longest substring without repeating characters (LC 3)
public int lengthOfLongestSubstring(String s) {
    Map<Character, Integer> lastSeen = new HashMap<>();
    int maxLen = 0, left = 0;
    for (int right = 0; right < s.length(); right++) {
        char c = s.charAt(right);
        if (lastSeen.containsKey(c)) {
            left = Math.max(left, lastSeen.get(c) + 1);
        }
        lastSeen.put(c, right);
        maxLen = Math.max(maxLen, right - left + 1);
    }
    return maxLen;
}
```

### Pattern 5: Prefix Sum with HashMap

```java
// Subarray sum equals k (LC 560)
public int subarraySum(int[] nums, int k) {
    Map<Integer, Integer> prefixCount = new HashMap<>();
    prefixCount.put(0, 1);
    int sum = 0, count = 0;
    for (int num : nums) {
        sum += num;
        count += prefixCount.getOrDefault(sum - k, 0);
        prefixCount.merge(sum, 1, Integer::sum);
    }
    return count;
}
```

## Useful Methods Reference

| Method | Description | Returns |
|--------|-------------|---------|
| `put(k, v)` | Insert/update | Previous value or null |
| `get(k)` | Retrieve value | Value or null |
| `getOrDefault(k, d)` | Safe get | Value or default |
| `containsKey(k)` | Key existence | boolean |
| `containsValue(v)` | Value existence | boolean |
| `remove(k)` | Delete entry | Removed value or null |
| `putIfAbsent(k, v)` | Insert if missing | Existing value or null |
| `merge(k, v, fn)` | Conditional update | New value |
| `compute(k, fn)` | Always recompute | New value |
| `computeIfAbsent(k, fn)` | Create if missing | Value |
| `replace(k, v)` | Update existing | Old value or null |
| `entrySet()` | Key-value pairs | Set\<Entry\> |
| `keySet()` | All keys | Set\<K\> |
| `values()` | All values | Collection\<V\> |
| `size()` | Entry count | int |
| `isEmpty()` | Empty check | boolean |
| `clear()` | Remove all | void |

## Common Problems

### Problem 1: Two Sum ⭐⭐⭐⭐⭐

**LeetCode 1** | Easy

```java
public int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> seen = new HashMap<>();
    for (int i = 0; i < nums.length; i++) {
        int need = target - nums[i];
        if (seen.containsKey(need)) return new int[]{seen.get(need), i};
        seen.put(nums[i], i);
    }
    return new int[]{};
}
// Time: O(n) | Space: O(n)
```

### Problem 2: Group Anagrams ⭐⭐⭐⭐

**LeetCode 49** | Medium

```java
public List<List<String>> groupAnagrams(String[] strs) {
    Map<String, List<String>> map = new HashMap<>();
    for (String s : strs) {
        char[] arr = s.toCharArray();
        Arrays.sort(arr);
        map.computeIfAbsent(new String(arr), k -> new ArrayList<>()).add(s);
    }
    return new ArrayList<>(map.values());
}
// Time: O(n * k log k) | Space: O(n * k)
```

### Problem 3: Top K Frequent Elements ⭐⭐⭐⭐

**LeetCode 347** | Medium

```java
public int[] topKFrequent(int[] nums, int k) {
    Map<Integer, Integer> freq = new HashMap<>();
    for (int n : nums) freq.merge(n, 1, Integer::sum);

    PriorityQueue<Integer> heap = new PriorityQueue<>(
        Comparator.comparingInt(freq::get)
    );
    for (int key : freq.keySet()) {
        heap.offer(key);
        if (heap.size() > k) heap.poll();
    }

    int[] result = new int[k];
    for (int i = k - 1; i >= 0; i--) result[i] = heap.poll();
    return result;
}
// Time: O(n log k) | Space: O(n)
```

## Key Takeaways

- `HashMap` gives **O(1) average** get/put/remove; worst case O(n) with many collisions.
- Always use `getOrDefault` or `containsKey` before `get` to avoid NullPointerException.
- `merge(k, 1, Integer::sum)` is the cleanest way to build a frequency map.
- `computeIfAbsent` is ideal for building **map of lists** (grouping patterns).
- Iteration order is **not guaranteed** — use `LinkedHashMap` for insertion order, `TreeMap` for sorted order.
- Java 8+ treeifies buckets with > 8 entries, keeping worst-case at O(log n).
