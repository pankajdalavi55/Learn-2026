## When to Use Each Style

### Ultra-Formal

**Use for:**
- Legal matters
- Executive presentations
- Official documentation
- Compliance/audit communications
- Performance improvement plans
- Termination/disciplinary matters

**Characteristics:**
- Third person where appropriate
- Complete sentences, perfect grammar
- No contractions (do not vs don't)
- No casual language
- Formal titles (Dr., Mr., Ms.)

**Example:**
```
Dear Mr. Thompson,

This letter serves as formal notification of the database migration 
scheduled for January 15, 2026. Per our service agreement, we are 
providing 30 days' advance notice.

The migration is expected to be completed within the designated 
maintenance window of 2:00 AM to 6:00 AM EST. 

Should you have any concerns, please contact our support team at 
support@company.com.

Respectfully,
[Your Full Name]
[Your Title]
```

### Formal

**Use for:**
- Initial contact with external stakeholders
- Client-facing communication
- Executive updates
- Official announcements
- Performance reviews
- Incident reports

**Characteristics:**
- Professional but not stiff
- Contractions OK in moderation
- Clear structure
- Respectful tone
- Proper greetings/closings

**Example:**
```
Subject: Q4 System Performance Report

Hello Leadership Team,

I'm pleased to share our Q4 system performance results. 

Highlights:
- 99.95% uptime (exceeded our 99.9% SLA)
- Average response time: 180ms (20% improvement)
- Zero critical incidents

The full report is attached with detailed metrics and analysis.

Please let me know if you have questions or need additional data.

Best regards,
Alex Chen
Senior Backend Engineer
```

### Semi-Formal

**Use for:**
- Manager communication
- Cross-team collaboration
- Stakeholder updates
- First-time contacts internally
- Most professional email
- Team-wide announcements

**Characteristics:**
- Friendly but professional
- Contractions common
- Can use emoji sparingly
- Conversational but respectful
- Clear and structured

**Example:**
```
Subject: Deploy Schedule for This Week

Hi team,

Here's our deploy schedule for the week:

Tuesday 2 PM - User dashboard updates
Thursday 6 PM - Database migration (expect 30 min downtime)

Please:
- Code freeze Tuesday at noon
- Be available Thursday evening for rollback if needed
- Update the deploy sheet if you have concerns

Let me know if this timing doesn't work.

Thanks,
Alex
```

## Matching Context

### Consider These Factors

1. **Relationship:** How well do you know them?
2. **Medium:** Email vs Slack vs in-person
3. **Topic:** Social vs technical vs sensitive
4. **Audience:** Individual vs group vs public
5. **Company culture:** Startup vs enterprise
6. **Seniority:** Peer vs manager vs executive
7. **Internal vs external:** Employee vs client

### Decision Matrix

| Factor | Formal | Informal |
|--------|--------|----------|
| **Relationship** | New/distant | Close/established |
| **Medium** | Email | Slack/chat |
| **Topic** | Sensitive/official | Routine/social |
| **Audience** | Large/public | Small/private |
| **Culture** | Corporate | Startup |
| **Direction** | Upward/external | Peer/downward |

## Transitioning Between Styles

### Starting Formal, Moving Informal

When building new relationships, start formal and mirror their style.

**Example progression:**

**First email:**
```
Dear Ms. Rodriguez,

I hope this email finds you well. I'm reaching out regarding...

Best regards,
Alex Chen
```

**After they respond casually:**
```
Hi Maria,

Thanks for the quick response! That makes sense...

Best,
Alex
```

**After several exchanges:**
```
Hey Maria,

Quick follow-up on that...

Thanks!
Alex
```

### Reading Social Cues

**Signs to stay formal:**
- They use your last name
- Formal closings (Sincerely, Respectfully)
- Longer, structured messages
- No emoji or casual language

**Signs you can be more casual:**
- They use your first name
- Casual closings (Thanks, Cheers)
- Shorter messages
- They use emoji

## Industry & Company Culture

### Startup Culture

Generally more informal:
- First names, even for C-suite
- Emoji common
- Casual dress, casual communication
- Slack > Email
- Speed > formality

**Example:**
```
@channel heads up - pushing a hotfix in 5 min ğŸš€
if your local env breaks, just pull latest from main
```

### Enterprise Culture

Generally more formal:
- Titles sometimes used
- Email > Chat
- Structured communication
- More meetings
- Process-oriented

**Example:**
```
Subject: Notification - Hotfix Deployment

Team,

Please be advised that a hotfix will be deployed to production 
at 3:00 PM today to address the payment processing issue.

Action required: Please pull the latest changes from the main 
branch after deployment to sync your local environment.

Deployment notes: [link]

Regards,
Alex Chen, Senior Engineer
```

### Finding Your Company's Style

**Observe:**
- How does leadership communicate?
- What's the email tone in your inbox?
- How do successful people communicate?
- What's the Slack channel vibe?

**When in doubt:** Match your manager's style.

## Global/Cultural Differences

### What's "normal" varies by culture:

**United States:** 
- Generally informal
- First names common
- Direct communication

**United Kingdom:**
- More formal than US
- Understated language
- Indirect sometimes

**Germany:**
- Can be quite direct
- Formal initially
- Title usage common

**Japan:**
- Very formal
- Hierarchical
- Indirect refusals

**When working globally:**
- Start more formal
- Be extra clear (no idioms)
- Be patient with language barriers
- Don't assume your normal is their normal
