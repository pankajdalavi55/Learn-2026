## üí° Tips for Success

1. **Understand First, Code Later**: Don't rush to code. Understand the problem completely.
2. **Start Simple**: Begin with brute force, then optimize.
3. **Time Yourself**: Practice with time constraints.
4. **Review Solutions**: After solving, review optimal solutions.
5. **Maintain a Journal**: Note down patterns and tricks.
6. **Consistent Practice**: 2-3 hours daily is better than 10 hours once a week.
7. **Don't Just Read**: Implement everything yourself.
8. **Test Edge Cases**: Always test with boundary conditions.

## üìå Quick Reference

### Time Complexity Cheat Sheet
| Operation | Best | Average | Worst |
|-----------|------|---------|-------|
| Array Access | O(1) | O(1) | O(1) |
| Array Search | O(1) | O(n) | O(n) |
| Binary Search | O(1) | O(log n) | O(log n) |
| HashMap Get/Put | O(1) | O(1) | O(n) |
| BST Search | O(log n) | O(log n) | O(n) |

### Space Complexity Patterns
- Recursion: O(h) where h is recursion depth
- Dynamic Programming: Often O(n) or O(n¬≤)
- Graph Algorithms: O(V + E)
